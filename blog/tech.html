SQL Injection

Week 8

March 10th, 2016

SQL Injection, simply stated is a code injection (forced placement) technique that is used to attack applications that sources data, in which the malicious statements are inserted to gain access to that database. The use of the injection is the exploit security vulnerabilies to the application's software. The purpose of these attacks is to allow the attacker to gain access to the data, and tamper with the existing data, or simply destroying the data. The interesting think about SQL injection, is that it is a common issues with any database-driven websites. A study in 2012 shows that any site will be attacked by a SQL injection on average 4 times per month.

Here is a simple example from www.owasp.org of what is going on with an SQL injection:

    
In SQL:

    
      select id, firstname, lastname from authors
    
    
If one provided:

    
      Firstname:evil'ex
      Lastname: Newman
    
    
the query string becomes:

    
      select id, firstname, lastname from authors where forename = 'evil'ex' and surname ='newman'
    
    
the database will attempt to run this as:

    
      Incorrect syntax near il' as the datavase tried to excute evil.

At the core, the attack is accomplished by placing a meta character into data input to then place SQL commands in the control plane which did not exist there before. That being said, stopping attackers from accessing your database should be a top priority, as your database generally holds sensitive data that should not be open to anyone snooping about.
