Ruby Hashes and Javascript Objects

Week 7

March 5th, 2016

There are a number of similarities and differences when it comes to compairing Ruby hashes and Javascript's Objects. The main points to look at are what hashes and objects are. At their core they are simply collections of data, that have a lot of similarites when creating a hash or an object.

    
      # Ruby Hash (using symbol format)
        student = {
          first_name: "Mike",
          last_name: "Verthein",
          age: 28,
          city: "Chicago",
      }
      // Javascript Object
        var student = {
          first_name: "Mike",
          last_name: "Verthein",
          age: 28,
          city: "Chicago",
      }

However, we start seeing some differences when we want to access their values, and their their lebelled values. In Ruby we call the label a key, and in JavaScript, we call it a property, but the effect is essentially the same. The corresponding values (both Ruby and JavaScript are labelled the same) can be anything. In the examples above they are strings and integers, however values in both can also be arrays or other hashes or objects.

      
        # Ruby Hash:
          student[:age]
          => 28

        // Javascript object:
          student.age;
          => 28
